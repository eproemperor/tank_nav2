
## 插件 = 类 + 注册宏 + 配置文件

```cpp

// ✅ 是写一个类，实现特定接口
class MyPlanner : public SomeBaseClass {
public:
    // 必须实现基类要求的函数
    Path computePath() override { ... }  // 你的算法
    void configure() override { ... }    // 初始化
    void cleanup() override { ... }      // 清理
};

// 必须导出插件
PLUGINLIB_EXPORT_CLASS(MyPlanner, SomeBaseClass)
```

## 插件的三个必要组成部分

### 1️⃣ **实现基类接口** (必须继承某个基类)
```cpp
// 行为树插件
class GoBase : public BT::SyncActionNode {  // 必须继承
    BT::NodeStatus tick() override { ... }  // 必须实现
};

// 规划器插件  
class CustomPlanner : public nav2_core::GlobalPlanner {  // 必须继承
    Path computePath() override { ... }  // 必须实现
};
```

### 2️⃣ **注册宏** (让框架能找到你的插件)
```cpp
// 行为树插件注册
BT_REGISTER_NODES(factory) {
    factory.registerNodeType<GoBase>("GoBase");
}

// 规划器插件注册
PLUGINLIB_EXPORT_CLASS(CustomPlanner, nav2_core::GlobalPlanner)
```

### 3️⃣ **配置文件** (告诉框架加载哪个插件)
```yaml
# nav2_params.yaml
bt_navigator:
  ros__parameters:
    planner_plugin: "custom_planner/CustomPlanner"  # 指定你的插件
```
